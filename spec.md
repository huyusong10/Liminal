# Goal

在 `playground/habit-tracker/` 里做一个可直接打开和手动体验的单页习惯打卡 Web 应用。完成后，用户应该能一眼看懂今天该做什么，能新增和管理习惯，能为今天打卡，并能从主界面看到连续打卡状态与整体进展。

# Checks

### 首屏目标清晰
- When: 新用户第一次打开应用
- Expect: 用户能立刻理解这是一个习惯打卡产品，并且能在首屏找到“新增习惯”和“今天打卡”的主要入口
- Fail if: 首屏意图模糊、主要操作不明显，或用户需要四处寻找下一步

### 习惯管理可用
- When: 用户新增、编辑、删除一个习惯
- Expect: 每个操作都有直接可见的界面反馈，结果能立即反映在主界面上
- Fail if: 任一核心操作缺失、报错，或界面状态前后不一致

### 今日打卡流程闭环
- When: 用户把某个习惯标记为今天已完成，然后取消或重新调整
- Expect: 完成状态、今日统计和连续打卡展示会同步更新
- Fail if: 打卡动作没有改变可见状态，或相关统计没有一起变化

### 刷新后状态仍然存在
- When: 用户完成若干习惯操作后刷新页面
- Expect: 已创建的习惯和当前打卡状态仍然可见
- Fail if: 一刷新就丢失主要数据，或页面回到空白初始态

# Constraints

- 只在 `playground/habit-tracker/` 内新增或修改文件，不要改动其他现有项目代码。
- 优先做前端可见成果；除非确实必要，不要引入后端服务。
- 产物应当可以用静态方式打开，或通过一个非常轻量的本地静态服务预览。
- 保持实现聚焦在“习惯管理 + 今日打卡 + 连续状态展示”这条主路径，不要扩展成大型平台。
