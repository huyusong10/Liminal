# Loopora Contracts

These contracts sit below `HUMAN-SHAPED-LOOP.zh-CN.md` and `README.zh-CN.md`. They define what must stay true while the
implementation stays free to shrink.

| Boundary | Stable claim | Primary code | Verification | Not a contract |
| --- | --- | --- | --- | --- |
| Product shape | Loopora preserves task-scoped human judgment across long Agent work. The Loop, not a prompt pack or DAG engine, is the primary object. | `src/loopora/service*.py`, `src/loopora/workflows.py` | README checks, shared compiler/runtime contracts, verdict tests | Role count, exact copy, visual layout |
| Dual entry | Web composer and Agent Native are peer entry surfaces into the same Core. Web is full-function; Agent Native must not grow a second compiler. | `service_alignment.py`, `service_agent_adapters.py`, `bundles.py` | Web READY/run smoke, Agent `/loopora-gen -> /loopora-loop`, shared bad-bundle error | Web-only observer framing, adapter-specific prose |
| Compiler | The compiler turns task judgment into a reviewable and runnable Loop. Web dialogue, Agent candidate plans, YAML import/export, preview, and run creation all use the same bundle normalization and semantic lint. READY preview must reflect current canonical content. | `src/loopora/bundles.py`, `service_bundle_assets.py`, `service_alignment.py` | One shared success path and one shared failure path across Web/Agent | UI summaries, traceability aliases, stale validation |
| Bundle | Bundle is a Web-operable exchange file: generate, edit, preview, import, export, replace, delete. It is not a long-term catalog or governance object. UI projections such as summaries, traceability aliases, or diagnostics are derived views. | `bundles.py`, bundle routes/templates | YAML import/export, preview, run handoff, absence of lineage/governance-card defaults | Lineage, revision identity, cards, dashboard metrics |
| Runtime | Run status and Loop verdict are separate. Evidence refs, coverage, and GateKeeper verdict decide whether the task is proven. | `service_run_*`, `service_agent_native.py`, `run_takeaways.py` | verdict/status tests, evidence ledger checks, run detail smoke | Process success as task proof, full artifact shape |
| Workflow | The default user model is linear and explainable: `Builder -> optional Inspector/Guide -> GateKeeper`. Advanced parallel/control fields are compatibility or expert-only surfaces. | `workflows.py`, workflow runtime | default preset tests, minimal advanced negative/compat checks | Parallel/control as default fixture or journey path |
| Agent Native | Agent Native lets the current Coding Agent remain the execution subject. `/loopora-gen` and `/loopora-loop` keep the current host Agent on the same reviewed Loop. Templates/tests inherit host model/provider defaults unless explicitly supplied. | `agent_adapters.py`, `service_agent_native.py`, CLI agent commands | adapter entry tests, capsule/submit contract, no hardcoded model/effort, opt-in real-agent probe | Nested provider CLI execution |
| Web Composer | Web supports conversation, compile, READY review, edit/improve, YAML import/export, create run, and run/evidence/verdict observation. | Web routes/templates/static pages | route availability, READY preview, create-run, run detail journey | CSS, DOM hierarchy, static wording, pixel layout |
