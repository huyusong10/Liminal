Getting Started with Loopora

Learn how to use Loopora for long, evidence-driven agent work.

What is Loopora?

Loopora is a local-first loop orchestration platform for long, evidence-driven agent work. It exists because some end-to-end tasks keep pulling humans back in to confirm, judge, and steer the work.

Instead of humans stepping in after every meaningful step, Loopora turns those repeated checkpoints into a structured workflow so humans can step in less often, but step in with better evidence when they do.

The Workflow

Loopora uses a four-role workflow that mirrors how experienced teams actually work:

Builder
→
Inspector
→
GateKeeper
→
Guide
🔨 Builder
Moves the workspace forward
🔍 Inspector
Reopens evidence from updated state
🛡️ GateKeeper
Decides if work crossed the line
🧭 Guide
Redirects when work stalls
Choosing an Orchestration

Loopora provides five built-in orchestrations for different types of tasks. Choose the one that matches your situation:

Build First
For initial implementations and getting things working end-to-end
Inspect First
For diagnosing regressions and understanding what broke
Triage First
For multi-issue diagnosis and narrowing down blockers
Repair Loop
For iterative repairs that need inspection between fixes
Benchmark Loop
For metrics-driven work gated by benchmark results
Writing Good Specs

Loopora specs stay short. They only need four sections:

# Task Describe what needs to be done. Be specific about the scope and what success looks like. # Done When - [ ] Criterion 1: Specific, measurable outcome - [ ] Criterion 2: Another specific outcome - [ ] Criterion 3: Final success condition # Guardrails - Constraint 1: What not to do or avoid - Constraint 2: Important boundaries or limits # Role Notes Builder: Specific guidance for the Builder Inspector: What the Inspector should focus on GateKeeper: Approval criteria and standards
