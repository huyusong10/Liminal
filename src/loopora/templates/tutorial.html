当一次 Builder 还不够，而且下一步得看新证据时，再用 Loopora。 Use Loopora when one Builder pass is not enough and the next move depends on fresh evidence.

Loopora 不是为了让 agent 一直跑下去，而是为了让 Builder、Inspector、GateKeeper 和 Guide 围绕新证据，一轮轮把任务收敛下来。 Loopora is not for keeping an agent running longer. It is for helping Builder, Inspector, GateKeeper, and Guide converge round by round around fresh evidence.

核心精神Core idea

为什么要 loopWhy loop

只有当这一轮行动会改变下一步判断时，loop 才有意义。否则只是让 Builder 围着旧假设越跑越偏。 A loop only matters when this round should change the next decision. Otherwise Builder is just drifting farther on the same stale assumption.

角色分工Role story

Builder 负责推进改动，Inspector 负责拿证据，GateKeeper 负责判断能不能过，Guide 只在卡住或明显走偏时介入。 Builder moves the work forward, Inspector gathers evidence, GateKeeper decides whether it passes, and Guide only steps in when the loop stalls or starts drifting.

什么时候别用When not to use it

如果只是改小文案、补一个小按钮，或者一轮就能安全做完的功能，就别开 loop。那种任务直接单次执行更省事。 If it is just a copy tweak, a tiny button, or a feature slice one agent can finish safely in a single pass, skip the loop. A one-shot run is simpler.
