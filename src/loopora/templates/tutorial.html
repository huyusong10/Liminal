AI Agent 已经能做，为什么还要用 Loopora？ If an AI Agent can already do the work, why use Loopora?

先反过来问：一位 AI Agent 加一轮人工 review 够不够？如果够，就别用 Loopora。只有当人类会在每轮之后反复回来判断证据、裁决是否过线、决定下一步方向时，才值得把这次任务的协作姿态编译成 bundle，再导入循环运行。 Ask the opposite question first: would one AI Agent pass plus one human review be enough? If yes, skip Loopora. Use it when humans would otherwise return after each round to judge evidence, decide whether the line was crossed, and choose the next move. That is when the task posture should be compiled into a bundle and imported as a loop.

先问这四个问题Ask these four questions first

你要省下的是产出，还是判断？Are you saving output, or judgment?

如果只是让 AI Agent 多写一次代码，Loopora 没有必要。它真正省下的是人类反复回场做的判断：证据够不够、是否假完成、该继续还是转向。 If the only goal is another code pass, Loopora is unnecessary. It saves the repeated human judgment loop: is the evidence real, is this fake done, should the next round continue or redirect?

姿态为什么不能只写成 prompt？Why is posture not just a prompt?

因为 prompt 只会影响说法。可运行的姿态必须同时落到三层：spec 冻结契约，角色定义塑造判断风格，workflow 决定判断时机。 Because a prompt only changes the wording. Runnable posture needs three surfaces: the spec freezes the contract, role definitions shape judgment style, and the workflow decides when judgment happens.

为什么不先手工编排？Why not hand-build the flow first?

多数人一开始没有现成的 spec / roles / workflow 地图，只有“这次不能糊弄”“我更信证据”这类感觉。先用外部 AI Agent + Skill 对齐成 working agreement，再编译成单文件 YAML bundle。 Most people do not start with a ready spec / roles / workflow map. They start with signals like "this cannot be sloppy" or "evidence matters more here." Use an external AI Agent + Skill to align that into a working agreement, then compile a single YAML bundle.

什么时候别开 loop？When should you not open a loop?

如果下一轮不会产生新证据，或者一次人工 review 就能定案，就别开 loop。没有新证据的循环不是收敛器，只会漂移。 If another round will not create new evidence, or one human review can settle it, do not open a loop. A loop without new evidence is not convergence, just drift.

如果不用 Loopora，人类会回来几次？ Without Loopora, how many times would humans return?

如果答案是“一次”，别开 loop。如果答案是“每轮都要回来判断”，才继续往下选流程形状。 If the answer is "once", skip the loop. If the answer is "after every round", continue and choose the workflow shape.
