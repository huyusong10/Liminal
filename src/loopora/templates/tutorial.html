从 human-in-the-loop 到 human-shaped loop From human-in-the-loop to human-shaped loop

Loopora 把未来任务中人类本来要反复做的纠偏、质疑、取证、验收和阻断，提前编译成一个可运行的 Loop。模型学习通用能力，Loop 继承当前任务的判断方式。 Loopora moves the human corrections, doubts, evidence demands, acceptance calls, and blockers that would happen later into a runnable Loop shaped before execution. The model learns general capability; the Loop inherits task-scoped judgment.

先问这四个问题Ask these four questions first

你要省下的是产出，还是判断？Are you saving output, or judgment?

如果只是让智能体多做一轮，简单循环就够了。Loopora 真正节省的是人类反复回场做的判断：证据够不够、是否假完成、该继续还是转向。 If the only goal is another Agent pass, a simple loop is enough. Loopora saves the repeated human judgment loop: is the evidence real, is this fake done, should the next round continue or redirect?

姿态为什么不能只写成提示词？Why is posture not just a prompt?

因为提示词只会影响说法。可运行的判断力必须同时落到三层：Loop 契约冻结边界，角色定义塑造判断视角，流程决定判断时机。 Because a prompt only changes the wording. Runnable judgment needs three surfaces: the spec freezes the contract, role definitions shape judgment views, and the workflow decides when judgment happens.

为什么不让模型直接学？Why should the model not just learn this?

这次任务的判断有局部性、时效性和可争议性，不该静默写进模型权重。Loop 让它保持显式、可预览、可修改、可丢弃。 This task's judgment is local, temporary, and debatable. It should not disappear into model weights. A Loop keeps it explicit, previewable, editable, and disposable.

什么时候别开 Loop？When should you not open a loop?

如果下一轮不会产生新证据，或者一次人工复核就能定案，就别开 Loop。没有治理结构和新证据的 loop 不是收敛器，只是开盲盒。 If another round will not create new evidence, or one human review can settle it, do not open a loop. A loop without governance and new evidence is not convergence; it is a blind box.

先问：这件事真的需要 Loopora 吗？First ask: does this really need Loopora?

构建 · 检查 · 守门 · 引导
不要先选流程名。先问未来哪些人类纠偏会反复发生，再问能否把那些判断提前塑形成 Loop。 Do not start with workflow names. First ask which human corrections would repeat later, then ask whether those judgments can shape the Loop now.

如果不用 Loopora，人类会回来几次？ Without Loopora, how many times would humans return?

如果答案是“一次”，别开 Loop。如果答案是“每轮都要回来判断”，才继续往下选流程形状。 If the answer is "once", skip the loop. If the answer is "after every round", continue and choose the workflow shape.
