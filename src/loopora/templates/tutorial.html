当强 Agent 能做，但人类不该反复回来确认时，再用 Loopora。 Use Loopora when a strong agent could do the work, but humans should not have to keep coming back to confirm and redirect it.

Loopora 不是因为 agent 完全做不到，而是因为如果没有一条 `Builder → Inspector → GateKeeper → Guide` 的流程，人类就得在每个关键阶段反复回来检查、裁决和纠偏。它把这些高频确认点编排进 loop，减少人工介入次数。 Loopora is not here because the agent is helpless. It is here because, without a `Builder → Inspector → GateKeeper → Guide` flow, humans keep getting pulled back in to inspect, judge, and redirect the work after every meaningful step.

核心精神Core idea

减少人工反复介入Reduce repeated human check-ins

如果没有 loop，人类会在每个关键阶段回来问同一类问题：这轮到底算不算过、下一步该盯哪一层、是不是该转方向。Loopora 的价值，就是把这些反复出现的确认点交给流程本身处理。 Without a loop, humans keep returning to ask the same questions: did this round actually cross the line, which layer matters next, and should the work change direction? Loopora turns those repeated checkpoints into workflow structure.

角色分工Role story

Builder 推进实现，Inspector 重看证据，GateKeeper 替团队做通过与否的裁决，Guide 只在停滞或走偏时收窄下一步。它们一起替人类吸收本来要反复介入的监督工作。 Builder pushes the implementation, Inspector reopens the evidence, GateKeeper makes the pass-or-no-pass call, and Guide only narrows the next move when the loop stalls or drifts. Together they absorb the supervision work that would otherwise keep dragging humans back in.

什么时候别用When not to use it

如果一位强 Agent 加上一轮人工 review 就够了，就别开 loop。Loopora 只该用在那些本来会把人类反复拉回来确认和纠偏的长程任务上。 If one strong agent pass plus one human review is enough, skip the loop. Use Loopora only for long tasks that would otherwise keep pulling humans back in to confirm and redirect them.
