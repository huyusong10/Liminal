# 场景：用户通过对话生成 bundle，预览后创建并运行

**目标**：确认新用户可以只描述需求，就从 Bundle 对话页完成对齐、READY 预览、物化 loop 和启动 run 的完整闭环。

**前置**：本机至少有一个可用的 Agent 执行器配置；目标 workdir 是服务端可访问的真实项目目录。

**步骤**：
1. 打开 Bundle 创建页，确认首屏以需求输入为主，并且可以选择 CLI 工具与目标 workdir。
2. 填写一个真实需求并发送，确认页面展开为对话视图。
3. 在 Agent 执行期间观察实时状态，确认页面会持续表达“正在思考 / 正在执行”，而不是像卡死。
4. 等待 session 进入 READY，确认系统不是只展示 Agent 的口头结论，而是已经写出并通过硬校验的 bundle。
5. 查看统一预览，确认任务契约、协作角色、执行流程图和源 YAML 都来自同一份 READY bundle。
6. 在流程图节点上悬停或键盘聚焦，确认节点本体不漂移，并通过浮层看到角色概要。
7. 点击创建循环并运行，确认内部仍走 bundle 生命周期，随后跳转到 run 详情页。
8. 等待 run 进入终态，确认新创建的 bundle、loop 与 run 互相关联，且后续可从各自详情页继续查看。

**预期**：对话生成路径不要求用户理解 Skill 安装、手写角色或手工编排；READY 只由硬校验后的 bundle 决定；创建和运行沿用现有 bundle 与 run 生命周期；流程图预览稳定可读。
