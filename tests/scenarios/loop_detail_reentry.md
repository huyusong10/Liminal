# 场景：用户回到 loop 详情页重新理解当前 loop 状态

**前置**：已经存在一条 loop，且至少跑过一次；用户想先看 loop 本身的配置和最新概况，再决定是否进入某次 run

**步骤**：
1. 打开任意一条已存在 loop 的详情页
2. 先阅读“配置”面板，确认能快速看到 orchestration、completion mode、轮次间隔、最大迭代和 spec 路径
3. 继续查看“原始 Spec”面板，确认长路径不会把布局撑坏，spec 内容在固定预览区内仍然可滚动浏览
4. 查看“运行历史”面板，确认每条 run 卡片都能看出状态、迭代和更新时间，并能继续点击进入 run 详情
5. 如果页面存在“最近摘要”面板，确认它能在不离开当前页的情况下给出最近一次 run 的状态、结论和一句话摘要

**预期结果**：
- loop 详情页能够作为 loop-level 的“基地页”，先帮助用户理解当前 loop 再进入某次 run
- 原始 spec 面板在长路径和长内容下仍然保持可读，不会无限撑高整页
- 运行历史保持明显的可点击反馈，状态和更新时间都容易扫读
- 最近摘要与运行历史分工清晰：前者负责快速判断，后者负责进入具体 run
