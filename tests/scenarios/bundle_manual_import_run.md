# 场景：用户手工导入已有方案文件，预览后运行

**目标**：确认已经从外部智能体、技能或团队共享文件得到方案文件的用户，可以在手动编排页先预览，再导入并启动运行。

**前置**：已有一份符合当前 bundle 契约的单文件 YAML；其中 `loop.workdir` 指向服务端可访问的真实项目目录。

**步骤**：
1. 打开手动编排页并进入方案文件导入区域。
2. 粘贴 YAML 或填写方案文件路径，然后先执行预览。
3. 确认预览不会创建方案包、Loop 或运行，只做校验和投影展示。
4. 查看 Loop 契约、协作角色、执行流程图和源文件，确认它们与待导入方案文件一致。
5. 在流程图节点上悬停或键盘聚焦，确认节点位置稳定，并能看到角色摘要浮层。
6. 选择导入后立即运行，提交导入。
7. 确认导入会物化方案包、角色定义、流程编排和 Loop，而不是绕过现有方案包生命周期直接创建运行。
8. 等待运行进入终态，确认从运行、Loop 和方案包三个入口都能重新找到这次导入的关系。

**预期**：已有方案文件路径在手动编排页完成导入，但仍使用与对话生成 READY 相同的信息结构预览；预览是只读校验，导入才产生资产；导入并运行后的资产关系完整、可追踪。
