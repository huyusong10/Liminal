# 场景：用户手工导入已有 bundle，预览后运行

**目标**：确认已经从外部 Agent、Skill 或团队共享文件得到 YAML bundle 的用户，可以在同一个 Bundle 页面先预览，再导入并启动 run。

**前置**：已有一份符合当前 bundle 契约的单文件 YAML；其中 `loop.workdir` 指向服务端可访问的真实项目目录。

**步骤**：
1. 打开 Bundle 创建页并进入已有 YAML / 文件区域。
2. 粘贴 YAML 或填写 bundle 文件路径，然后先执行预览。
3. 确认预览不会创建 bundle、loop 或 run，只做校验和投影展示。
4. 查看任务契约、协作角色、执行流程图和源 YAML，确认它们与待导入 bundle 一致。
5. 在流程图节点上悬停或键盘聚焦，确认节点位置稳定，并能看到角色摘要浮层。
6. 选择导入后立即运行，提交导入。
7. 确认导入会物化 bundle、角色定义、流程编排和 loop，而不是绕过现有 bundle 生命周期直接创建 run。
8. 等待 run 进入终态，确认从 run、loop 和 bundle 三个入口都能重新找到这次导入的关系。

**预期**：已有 YAML 路径和对话生成路径共享同一套预览体验；预览是只读校验，导入才产生资产；导入并运行后的资产关系完整、可追踪。
