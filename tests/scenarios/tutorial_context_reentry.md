# 场景：用户回到教程页重新判断 bundle 与手动资产的关系

**目标**：确认教程页在长内容阅读场景下仍然清楚说明默认循环方案路径，以及手动角色 / 编排 / loop 编辑为什么只是 expert mode。

**前置**：用户已经用过一次 Loopora，但隔了一段时间后回来，想重新确认“是重新生成 bundle，还是手动改某个资产”。

**步骤**：
1. 打开教程页，先阅读顶部反问式标题，确认页面先问“AI Agent 已经能做，为什么还要用 Loopora”。
2. 确认页面用连续问题说明：用户真正要省下的是每轮后的人工判断，而不是让 AI Agent 多写一次代码。
3. 确认页面说明姿态不能只写成 prompt，而要通过 `spec`、角色定义和 workflow 共同承载。
4. 确认页面明确说明 working agreement 是对齐阶段的临时确认物，最终导入的是 YAML bundle。
5. 确认页面说明 bundle 会同时物化 spec、角色定义、workflow 和 loop，不需要用户先手工拆成多个资产。
6. 从右侧资源与设置菜单进入“工具与 Skill”“手动创建”等入口，并从顶层“新建任务”进入对话生成路径，确认已有 YAML 导入仍回到手动专家路径。
7. 浏览下方 5 个 workflow 样例，确认它们仍然用于解释姿态如何影响流程形状，而不是要求新用户先手写流程。

**预期**：教程页即使在长篇阅读场景下也不会丢失主线；用户能优先选择 bundle revision 路径，并在确实知道要改哪个 surface 时再进入手动专家模式。
