# 场景：用户首次从网络模式入口进入 Loopora

**前置**：Loopora 以网络模式暴露，浏览器当前没有保存认证 cookie；用户浏览器语言可能是英文，也可能是中文

**步骤**：
1. 直接访问未带 token 的首页地址。
2. 观察认证引导页是否说明当前实例处于网络模式，并明确指出需要 token。
3. 检查页面是否给出 URL 查询参数示例，以及 `Authorization`、`X-Loopora-Token` 这些可用入口。
4. 用读屏或辅助技术聚焦认证卡片，确认品牌 logo 只是装饰，不会额外读出一段固定英文品牌文案来打断主提示。
5. 调整浏览器语言偏好顺序和权重，例如让 `zh-CN` 先出现但权重更低、`en-US` 后出现但权重更高，再确认页面首屏语言会跟随真正的首选请求 locale 初始化，而不是只按出现顺序选语言。
6. 在地址中补上有效 token 再访问一次，确认可以正常进入应用。

**预期结果**：
- 未授权访问不会误进主界面，而是先落到明确的认证引导页
- 认证引导页能说明 token 的几种传递方式，减少用户猜测成本
- 认证页里的品牌图不会作为一段固定英文额外被读屏播报，主信息仍然聚焦在认证提示本身
- 页面语言不会被硬编码锁死在单一语言，且会按 `Accept-Language` 的真实优先级初始化，而不是被低优先级 fallback 语言或单纯的出现顺序误导
- 补上有效 token 后可以成功进入，并建立后续访问所需的认证状态
