# 场景：资源库、工作台卡片与专家编辑入口

**目标**：确认工作台、方案库、角色定义和流程编排这些资源入口能互相衔接，同时危险操作和主进入动作保持分离。

**前置**：

- 至少存在一条有 run 的 loop、一条无 run 的 loop、一个自定义角色和一个自定义编排。
- 页面可在中文或英文任一界面语言下打开。

**步骤**：

1. 从工作台打开有 run 的 loop 卡片主体，确认进入最值得继续查看的 run 或 loop 详情。
2. 返回工作台，分别触发同一卡片的查看 loop 与删除动作，确认它们不会被整卡点击吞掉。
3. 进入 loop 详情页，确认配置、spec、运行历史和最近摘要能帮助用户先理解 loop，再进入具体 run。
4. 进入角色定义与流程编排列表，确认自定义卡片主体可进入编辑页，删除按钮保持独立。
5. 新建或编辑编排，确认空白起点不会自动变成默认预设，步骤卡片、流程图和设置浮窗能同步表达同一份 workflow。
6. 在手动创建页打开 spec 编辑器，确认编辑、预览、保存和关闭围绕同一路径工作。

**预期**：

- 顶层导航保持“工作台 / 新建任务 / 方案库”的主心智，资源与专家编辑位于次级入口。
- 卡片主体、次级入口和危险操作拥有独立可达的交互目标。
- 角色、workflow、spec 的专家编辑仍回到同一套 loop / bundle 生命周期，不制造孤立资产。
