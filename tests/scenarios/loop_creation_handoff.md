# 场景：创建循环时优先对话生成 bundle，必要时再进入手动专家模式

**目标**：确认“创建循环”页已经把对话生成 bundle 作为默认新手路径，同时把已有 YAML / 文件导入放到手动专家路径。

**前置**：用户可以选择直接描述需求生成 bundle，也可以已经从外部 Agent + Skill 得到一份 YAML bundle；系统里也至少存在一个可选编排，供用户验证手动专家模式仍可用。

**步骤**：
1. 打开“创建循环”页，先浏览顶部说明，确认默认路径是“对话生成 Bundle”。
2. 进入 Bundle 对话页，确认首屏只面向描述需求、选择 workdir 和 Agent，不出现已有 YAML 导入表单。
3. 点击“安装对齐 Skill”，确认能回到工具页安装 repo-local Skill；返回创建页。
4. 点击“管理已有 Bundles”，确认能进入 bundle 管理页查看、导出、派生或整包删除已有 bundle；返回创建页。
5. 点击“手动专家模式”锚点，确认手动 loop 表单仍然可用，并且已有 bundle YAML / 文件导入入口也在这里。
6. 在手动表单里用键盘或读屏聚焦字段旁的 tips 按钮，确认这些帮助入口不会只读出一个 `i`，而是会提示当前语言下的说明语义。
7. 填写 spec 路径并点击右侧“编辑”，确认会弹出共享 Markdown workbench，并默认先停留在源码编辑态，而不是一上来就把编辑区和预览区并排塞满。
8. 在编辑弹窗里确认头部标题说明、校验状态、预览/保存/关闭操作保持清晰分栏；修改内容后点击“保存”会写回同一路径的磁盘文件，点击“预览”才会切到渲染结果。

**预期**：创建循环页的默认主任务是对话生成 bundle 并物化可运行 loop；手动表单仍然完整，并承载已有 YAML / 文件导入。tips 与 spec 编辑器继续保持可访问、可保存、可预览，并且提交读取的内容与磁盘文件保持一致。
