# 场景：创建循环时优先导入 bundle，必要时再进入手动专家模式

**目标**：确认“创建循环”页已经把 bundle 导入作为默认路径，同时保留手动 loop 表单供熟悉内部资产的用户使用。

**前置**：用户已经从外部 Agent + Skill 得到一份 YAML bundle；系统里也至少存在一个可选编排，供用户验证手动专家模式仍可用。

**步骤**：
1. 打开“创建循环”页，先浏览顶部说明，确认默认路径是“导入 Bundle 创建循环”。
2. 在 bundle 导入表单中确认可以选择路径或粘贴 YAML，并可选择导入后立即运行。
3. 点击“安装对齐 Skill”，确认能回到工具页安装 repo-local Skill；返回创建页。
4. 点击“管理已有 Bundles”，确认能进入 bundle 管理页查看、导出、派生或整包删除已有 bundle；返回创建页。
5. 点击“手动专家模式”锚点，确认手动 loop 表单仍然可用，并且明确服务于已经准备好 spec、角色定义和流程编排的用户。
6. 在手动表单里用键盘或读屏聚焦字段旁的 tips 按钮，确认这些帮助入口不会只读出一个 `i`，而是会提示当前语言下的说明语义。
7. 填写 spec 路径并点击右侧“编辑”，确认会弹出共享 Markdown workbench，并默认先停留在源码编辑态，而不是一上来就把编辑区和预览区并排塞满。
8. 在编辑弹窗里确认头部标题说明、校验状态、预览/保存/关闭操作保持清晰分栏；修改内容后点击“保存”会写回同一路径的磁盘文件，点击“预览”才会切到渲染结果。

**预期**：创建循环页的默认主任务是导入 bundle 并物化可运行 loop；手动表单仍然完整，但不会和推荐路径抢主入口。tips 与 spec 编辑器继续保持可访问、可保存、可预览，并且提交读取的内容与磁盘文件保持一致。
