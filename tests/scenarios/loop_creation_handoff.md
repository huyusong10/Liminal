# 场景：新建任务时优先对话生成循环方案，必要时再进入手动专家模式

**目标**：确认顶层“新建任务”直接进入循环方案对话，同时把已有 YAML / 文件导入放到右侧资源菜单里的手动专家路径。

**前置**：用户可以选择直接描述需求生成循环方案，也可以已经从外部 Agent + Skill 得到一份 YAML bundle；系统里也至少存在一个可选编排，供用户验证手动专家模式仍可用。

**步骤**：
1. 点击顶层“新建任务”，确认直接进入循环方案对话页，而不是先落到内部概念选择页。
2. 在对话页确认首屏只面向描述需求、选择 workdir 和 Agent，不出现已有 YAML 导入表单。
3. 点击顶层“方案库”，确认能进入方案库查看、导出、派生或整包删除已有方案；返回工作台。
4. 打开右侧资源与设置菜单，确认能进入“工具与 Skill”安装 repo-local Skill。
5. 从同一个菜单进入“手动创建”，确认手动 loop 表单仍然可用，并且已有 bundle YAML / 文件导入入口也在这里。
6. 在手动表单里用键盘或读屏聚焦字段旁的 tips 按钮，确认这些帮助入口不会只读出一个 `i`，而是会提示当前语言下的说明语义。
7. 填写 spec 路径并点击右侧“编辑”，确认会弹出共享 Markdown workbench，并默认先停留在源码编辑态，而不是一上来就把编辑区和预览区并排塞满。
8. 在编辑弹窗里确认头部标题说明、校验状态、预览/保存/关闭操作保持清晰分栏；修改内容后点击“保存”会写回同一路径的磁盘文件，点击“预览”才会切到渲染结果。

**预期**：新建任务的默认主任务是对话生成循环方案并物化可运行 loop；方案库承担长期方案管理；手动表单仍然完整，并承载已有 YAML / 文件导入。tips 与 spec 编辑器继续保持可访问、可保存、可预览，并且提交读取的内容与磁盘文件保持一致。
