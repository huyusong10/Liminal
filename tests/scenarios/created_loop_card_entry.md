# 场景：用户从工作台进入最近运行

**目标**：确认“工作台”里，loop 卡片主体本身就是进入详情的主入口，同时查看循环和删除按钮仍保持独立点击目标。

**前置**：至少存在一条正在运行中的 loop，以及一条还没有 run 的 loop。

**步骤**：
1. 先用中文界面语言打开首页，确认浏览器页签标题与页面里的“工作台 / 当前工作”区域在首帧就是中文，不会先落成英文。
2. 用键盘或读屏聚焦顶栏品牌入口与资源设置控件，确认“首页 / 主题切换 / 语言切换”等辅助标签也已经是当前语言，而不是仍然读出英文。
3. 再观察那张运行中的 loop 卡片，确认卡片主体、次级按钮和删除按钮是清晰分离的。
4. 在这张运行中的卡片前停留几秒，确认“运行中”强调保持稳定，不会出现周期性扫光、脉冲或让整页亮度跳动的闪烁感。
5. 点击这张卡片的正文空白区域，确认会直接进入最近一次 run 的详情页。
6. 返回首页，再点击同一张卡片里的“查看循环”，确认会进入 loop 详情页，而不是最近 run。
7. 再次返回首页，点击删除按钮，确认只触发删除流程，不会先跳转。
8. 对那张还没有 run 的 loop 重复点击卡片主体，确认这次会进入 loop 详情页而不是空白 run 页面。

**预期**：首页在首帧就保持 locale 一致，顶栏辅助标签不会残留英文；运行中卡片的强调保持在卡片内部且视觉稳定，不会制造周期性闪烁；卡片主体始终承担“进入当前最值得继续看的页面”这个主动作；次级按钮和删除按钮保持独立语义，不会被整卡点击行为吞掉。
