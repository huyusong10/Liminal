# 场景：新用户通过教程页理解循环方案创建路径

**前置**：首次进入 Loopora Web UI，还不了解 collaboration posture、循环方案、创建循环三者的关系。

**步骤**：
1. 分别用英文和中文打开“使用教程”页面，确认浏览器页签标题与页内主标题在首帧就和当前界面语言一致。
2. 先阅读顶部反问式标题，确认页面不是先下定义，而是先追问“AI Agent 已经能做，为什么还要用 Loopora”。
3. 继续阅读四张问题卡，确认页面把主线推进成：省下什么人工判断、为什么不能只写 prompt、为什么不先手工编排、什么时候别开 loop。
4. 确认教程说明 posture 由 `spec`、角色定义和 workflow 共同承载，不把姿态误导成单个 prompt。
5. 继续阅读决策树，确认用户先判断“如果不用 Loopora，人类会回来几次”，再判断哪条流程最能减少下一次人工回场。
6. 滚到行动入口，确认默认路径是进入 Web 对话生成循环方案；安装对齐 Skill 是外部 AI Agent 可选路径，手动编排被明确标为 expert mode。
7. 打开任意流程样例弹窗，确认样例仍帮助用户理解 workflow 如何承载姿态，而不是把流程名当成静态规则。

**预期结果**：
- 教程页的文档标题与正文主标题在首帧就保持 locale 一致，不会先闪回默认英文。
- 新用户不用先理解所有内部资产，也能知道默认应该从 Web 对话生成循环方案，再预览、创建并运行。
- 教程不会把 working agreement 描述成运行期资产；最终运行输入仍然落在 bundle 编译出的 `spec / roles / workflow`。
- 手动入口仍然清晰可见，但不会盖过循环方案对话的推荐路径。
