# 场景：从空白编排开始通过步骤卡片调整流程

**前置**：进入“流程编排”页面，当前没有正在编辑的未保存编排

**步骤**：
1. 打开“新建编排”页面，确认默认是空白工作台而不是自动带出某个预设
2. 在步骤工具条中选择任意一个起手模板并载入
3. 观察页面先展示循环实例图、下方再展示步骤卡片，并确认图和卡片会联动
4. 点击任意一张步骤卡片的空白区域，再点击图中的任意节点，确认当前步骤高亮会同步切换
5. 点击步骤卡片中的“设置”，在浮窗里修改一个步骤 ID，并为某个 Builder 步骤打开或确认“继承上一次 session”，再填入一条附加 CLI 参数，例如 `--verbose`
6. 调整至少一个步骤的顺序，再删除一个步骤
7. 确认被任何步骤都不再引用的角色快照会自动从当前编排里清理
8. 保存编排并重新进入编辑页

**预期结果**：
- 空白新建页不会自动回退成默认预设
- 载入模板后，循环实例图和步骤卡片保持同步，整体采用上图下卡片的布局
- 点击卡片空白区域即可切换当前角色，不需要只点序号
- 图中的节点点击后会同步定位并高亮到对应的步骤卡片
- 浮窗里的修改会回写到卡片摘要，关闭时不需要额外保存确认
- Builder 步骤可以显式控制是否继承自己上一轮的 session，并能额外附加 CLI 参数；这些配置只影响当前步骤，不会改写角色快照
- 步骤调整后，循环实例图顺序同步刷新
- 不存在“步骤里已经没有，但编排 JSON 里还残留”的孤儿角色
- 保存后的编排可以正常重新打开
